## План разработки (минимум для защиты)

Для защиты проектного семинара (16-29 апреля) нужна **работающая демо** и текст ВКР на 80-90%.

### MVP — что реально нужно показать:

1. **Загрузка скриншота** через веб-интерфейс
2. **OCR распознавание** текста на скриншоте
3. **Извлечение данных** (сумма, мерчант, дата) из распознанного текста
4. **Сохранение** транзакции в БД
5. **Отображение** списка транзакций/подписок в UI

### Что можно НЕ делать для MVP:
- Дообучение PaddleOCR (используем базовую модель, пишем в ВКР что "дообучение запланировано")
- Notification Service
- Redis
- S3 (храним файлы локально)
- API Gateway (FastAPI сам роутит)
- Rate limiting
- 100 одновременных пользователей
- Тесты на 70%

### Порядок разработки:

#### Этап 1: Backend скелет (1-2 дня)
```
backend/
├── main.py              # FastAPI app
├── config.py            # настройки
├── models.py            # SQLAlchemy модели (из ER-диаграммы)
├── schemas.py           # Pydantic схемы
├── database.py          # подключение к PostgreSQL
├── routers/
│   ├── auth.py          # регистрация, логин, JWT
│   ├── transactions.py  # загрузка изображений, CRUD
│   └── subscriptions.py # список подписок
├── services/
│   ├── ocr.py           # PaddleOCR wrapper
│   ├── parser.py        # извлечение суммы/мерчанта/даты из текста
│   └── matcher.py       # fuzzy matching с базой сервисов
└── requirements.txt
```

#### Этап 2: БД + модели (0.5 дня)
- Создать PostgreSQL базу
- Alembic миграции для таблиц: users, subscriptions, transactions, services, banks
- Засеять таблицу services (Яндекс Плюс, Netflix, Spotify, YouTube Premium, VK Музыка и т.д.)
- Засеять таблицу banks (Сбербанк, Тинькофф, Альфа, ВТБ + regex паттерны)

#### Этап 3: OCR модуль (1 день)
- Установить PaddleOCR
- Обёртка: принимает изображение → возвращает распознанный текст + confidence
- Парсер: regex для извлечения суммы (паттерны: "1 234,56₽", "1234.56 RUB", "Покупка 299р")
- Парсер: извлечение даты
- Парсер: извлечение названия мерчанта

#### Этап 4: Matching + классификация (0.5 дня)
- fuzzy matching мерчанта с таблицей services (rapidfuzz)
- Простая классификация: если сумма совпадает с known_prices сервиса → подписка, иначе → разовый

#### Этап 5: Frontend (1-2 дня)
```
frontend/
├── src/
│   ├── App.jsx
│   ├── pages/
│   │   ├── Login.jsx
│   │   ├── Upload.jsx        # загрузка скриншота
│   │   ├── Transactions.jsx  # история транзакций
│   │   └── Subscriptions.jsx # список подписок
│   ├── components/
│   │   ├── FileUpload.jsx
│   │   ├── TransactionCard.jsx
│   │   └── SubscriptionCard.jsx
│   └── api.js               # axios calls to backend
```

#### Этап 6: Docker + скринкаст (0.5 дня)
- docker-compose.yml (postgres + backend + frontend)
- Записать скринкаст: запуск → регистрация → загрузка скриншота → результат OCR → список подписок

#### Этап 7: Презентация (0.5 дня)
- 10-12 слайдов: проблема → аналоги → архитектура → демо-скриншоты → результаты → выводы

#### Этап 8: Программа для ЭВМ (0.5 дня)
- Заполнить заявку на регистрацию программы для ЭВМ в Роспатент
- Нужно: название, аннотация, язык программирования, ОС, объём кода

## Ключевые решения для ВКР текста

В ВКР уже описана микросервисная архитектура. В реализации делаем **монолит на FastAPI** — это проще и быстрее. В тексте пишем:

> "Для прототипа реализован монолитный вариант архитектуры. Разделение на микросервисы запланировано на этапе масштабирования."

PaddleOCR без дообучения. В тексте:

> "Базовая модель PaddleOCR показала точность X% на тестовой выборке. Дообучение на доменно-специфичном датасете запланировано как следующий этап."
